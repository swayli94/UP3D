# NACA0012 viscous reference — XFOIL 6.99 (M 0.5, Re 3e6, α = 2°)

Viscous boundary-layer reference (`δ*/c`, `Cf` surface distributions and
`cl`/`cd`/`cm`) for NACA0012 at M∞ = 0.5, Re = 3.0×10⁶ (chord-based),
α = 2°, generated by `generate_xfoil_reference.py` driving **XFOIL 6.99**
in batch mode. XFOIL is an *external* code (Drela, MIT) — not a repo
dependency, not Python, not derived from the solver it gates — which makes
it a suitable independent reference for the viscous-coupling track (V3).

## Run conditions (pinned)

- Geometry: `NACA 0012` inside XFOIL (245 buffer points), repaneled to
  **280 panel nodes** (`PPAR` → `N 280`).
- M∞ = 0.5, Re = 3.0e6, α = 2.0°, viscous mode (`OPER` → `VISC 3.0E6`,
  `MACH 0.5`), Ncrit = 9 (XFOIL default, e^N only active upstream of the
  trip), Newton iterations `ITER 200` (both cases converge in ~5
  iterations to rms ≈ 1e-5).
- Two forced-transition variants (`VPAR` → `XTR xt xb`):
  - **xtr005** — trip at x/c = 0.05 on both surfaces (mostly turbulent).
  - **xtr030** — trip at x/c = 0.30 on both surfaces. Note: on the upper
    surface the e^N criterion triggers natural transition at x/c = 0.2668,
    *ahead* of the 0.30 trip, so the upper trip is inactive (XFOIL polar
    reports Top_Xtr = 0.2668); the lower surface is tripped at 0.30 as
    commanded.

## Results (α = 2°)

| case   | cl     | cd      | cm     | δ*/c TE upper | δ*/c TE lower |
|--------|--------|---------|--------|---------------|----------------|
| xtr005 | 0.2691 | 0.00926 | 0.0011 | 0.00713       | 0.00477        |
| xtr030 | 0.2756 | 0.00717 | 0.0002 | 0.00543       | 0.00385        |

Sanity notes (2026-07-22, XFOIL 6.99 DP build):

- cl at α = 2° lands in the expected ≈ 0.23–0.27 ballpark for mostly
  turbulent NACA0012 at M 0.5 / Re 3e6 (xtr005: 0.2691, xtr030: 0.2756 —
  xtr030 is marginally above 0.27; both kept, they are self-consistent and
  bracket the range).
- cd for **xtr005 = 0.00926 is intentionally above** the 0.004–0.008
  laminar-ish band: tripping at 5% chord makes the boundary layer
  turbulent over ~95% of the chord, which costs friction drag (XFOIL
  decomposition: CDf = 0.00722, CDp = 0.00204). xtr030 (0.00717) sits
  inside the band. A free-transition cross-check (same M/Re/α, no XTR)
  gives cl = 0.2635, cd = 0.00588, i.e. between the two tripped values,
  as it must.
- Cf > 0 everywhere on both surfaces in both cases (no separation), with
  a clear laminar→turbulent Cf jump across the 0.05 trip in xtr005
  (≈ 2.7e-3 → 5.4e-3). δ*(x) is smooth (max |2nd difference| ≈ 1e-4,
  at the TE where point spacing is coarsest).
- The two variants differ materially aft of 0.3c: TE δ*/c (upper)
  0.00713 vs 0.00543 (−24%), TE Cf likewise.

## Files

- `delta_star_cf_alpha2_m05_xtr005.csv`, `..._xtr030.csv` —
  `x_c,surface,dstar_over_c,cf`; `surface` ∈ {`upper`,`lower`}, each
  block sorted LE → TE (x_c ascending). 140 stations per surface.
- `polar_summary.csv` — `case,m_inf,re,alpha_deg,x_tr_upper,x_tr_lower,
  cl,cd,cm`, one row per variant. `x_tr_*` are the *commanded* trips (for
  xtr030 the actual upper-surface transition is at 0.2668, see above).
- `inviscid_summary.csv` — `case,m_inf,alpha_deg,cl,cm`: the inviscid
  XFOIL point at the same M/α (cl = 0.2921, cm = −0.0027, via PACC polar
  accumulation + one-point ASEQ — the 6.99 inviscid batch prints no CL to
  stdout). Reference for "XFOIL's own viscous decrement" (GV3.1 Δcl):
  0.2921 − 0.2691 = 0.0230 (xtr005).
- `generate_xfoil_reference.py` — the generator (rerun to reproduce).

## XFOIL build recipe (binary is gitignored under `tools/xfoil/`)

Source tarball: <https://web.mit.edu/drela/Public/web/xfoil/xfoil6.99.tgz>
(XFOIL Version 6.99, © 2000 Drela/Youngren, GPL). Built 2026-07-22 with
gfortran 10.2.1 on Linux x86-64, double precision:

```sh
mkdir -p tools/xfoil && cd tools/xfoil
wget https://web.mit.edu/drela/Public/web/xfoil/xfoil6.99.tgz
tar xzf xfoil6.99.tgz
cd Xfoil/plotlib
cp config.make.gfortranDP config.make
# edit config.make: FFLAGS += -std=legacy -fallow-argument-mismatch
make                                        # -> libPlt_gDP.a
cd ../bin && cp Makefile_gfortran Makefile
# edit Makefile: FFLAGS and FFLOPT += -std=legacy -fallow-argument-mismatch
# edit ../src/xfoil.f: change 'IDEV = 1' to 'IDEV = 2' (PostScript-only
# plotting -> batch runs are headless, no X11 display needed)
make xfoil BINDIR=.                         # -> Xfoil/bin/xfoil
```

Build fixes vs. the stock tarball: (1) `-std=legacy
-fallow-argument-mismatch` for gfortran ≥ 10; (2) the `IDEV = 2` patch —
stock XFOIL calls `PLTINI` after every computed point and aborts with
"Cannot open display" on a headless machine; (3) `BINDIR=.` because the
stock Makefile installs into `/home/codes/bin`. `-lX11` links fine against
the system `/lib64/libX11.so`. The `pplot`/`pxplot` utilities were not
built (not needed).

Regenerate the CSVs (requires the binary above):

```sh
python cases/reference_data/naca0012_viscous_xfoil/generate_xfoil_reference.py
```

The script exits non-zero with build instructions if no `xfoil` binary is
found under `tools/xfoil/`. Each run executes in a temporary directory, so
XFOIL side files (`plot.ps`, the `NNN.bl` TS-diagnostic dumps) never touch
the repo.

Do not edit the CSVs by hand (agent-rules hard rule 6); regenerate with
the script if the run conditions ever change, and record why here.
