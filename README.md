# UP3D
Unstructured-mesh full-potential solver for 3D flows.

## Tooling and Software Requirements

The development plan in [docs/roadmap.md](docs/roadmap.md) is feasible on Linux
without a graphical desktop, as long as visualization checks are generated by
scripts (PNG/CSV artifacts) instead of manual GUI inspection.

### Required

- Python 3.11+
- pip (or uv/poetry; pip is the baseline)
- Git
- Gmsh CLI (`gmsh`) for mesh generation
- Core Python packages (solver and tests):
	- `numpy`
	- `scipy`
	- `numba`
	- `meshio`
	- `pyamg`
	- `pytest`
	- `matplotlib` (headless plotting)

### Recommended for headless visualization

- `pyvista` + `vtk` for off-screen 3D rendering (`PYVISTA_OFF_SCREEN=true`)
- ParaView batch tools (`pvpython` or `pvbatch`) if available on your machine

### Optional

- `jupyter` for exploratory notebooks (not required for roadmap gates)
- `ruff` and `mypy` for code quality checks

## Headless Linux Workflow

1. Generate meshes with Gmsh CLI and keep all preprocessing scriptable.
2. Run solver and validation gates via `pytest`.
3. Export validation figures and numeric summaries as artifacts (PNG + CSV).
4. Use those artifacts as gate evidence; do not require interactive GUI steps.

Suggested artifact convention:

- `artifacts/<gate_id>/*.png`
- `artifacts/<gate_id>/summary.csv`
