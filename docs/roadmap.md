## 0. Working rules

- A phase is complete only when its medium-mesh gate passes and the full coarse regression suite stays green.
- After any kernel change, run `pytest tests/test_v0_freestream.py` before any broader validation.
- Use SciPy/PyAMG for linear algebra, keep Numba kernels SoA-only, and validate against full-potential references before Euler.
- Keep state in git plus this file; update the progress ledger when a gate closes.
- Every visual gate must have a headless path: generate PNG/CSV artifacts by script (no GUI-only checks).

## 0.1 Headless Linux feasibility and tooling policy

The current plan is feasible on Linux without a graphical desktop, with one constraint:
all visualization checks must be script-driven and artifact-based.

Required execution modes:
- Mesh generation/inspection: use `gmsh` CLI (`-2/-3`, `-format msh4`) and scriptable quality reports.
- Solver/testing: run via `pytest` and Python CLIs only.
- Visualization artifacts: prefer `matplotlib` for plots and `pyvista` off-screen rendering; use `pvpython`/`pvbatch` if ParaView is installed headless.
- CI compatibility: every gate that references a figure should accept generated PNG + numeric CSV summary as evidence.

Non-goals for v1.0:
- No dependency on interactive ParaView GUI for gate closure.

## Track M вҖ” Mesh generation pipeline (parallel to solver track)

Deliverable: `meshgen/` вҖ” scripted Gmsh generation of wake-embedded (M0/M1)
and wake-free (M3/M4, for Track B) tet meshes, coarse/medium/fine families per
case.

**Track-wide convention.** Model the wake as a Gmsh *embedded surface in volume*
(`gmsh.model.mesh.embed`); node duplication is NOT done in Gmsh вҖ” it is done by
the solver-side preprocessor (`mesh/wake_cut.py`), keeping meshing vanilla
(hard rule 8). The wake-**free** families (M3/M4) drop the embed step entirely:
the wake sheet survives only as the source of a size field, so nothing in the
mesh topology knows the wake exists вҖ” exactly the form Track B's level set
consumes (dual-mesh rule, Track B working rules). Any wake-cut change reruns the
topology asserts on all meshes in `cases/meshes/` (hard rule 7).

### M0 вҖ” Single-layer extruded NACA0012 quasi-2D mesh вң“ (closed 2026-07-06, with P2)
**Deliverable:** **Single-layer extruded NACA0012 quasi-2D mesh.** Input: 2D triangulated airfoil mesh (Gmsh). Extrude exactly ONE cell layer in z between two parallel planes; both z-planes tagged `symmetry`, other surfaces tagged `wall`, `farfield`, `wake`. Prisms subdivided into 3 tets each with a **globally consistent rule**: on every shared quad face, the diagonal connects to the smallest global node index; preprocessor assert that every interior quad face is split identically from both sides (no hanging nodes / slivers). **Planar wake surface embedded** from TE to far field (`gmsh.model.mesh.embed`; node duplication stays solver-side, unchanged). With a single layer there is exactly one spanwise station, so О“ is a single scalar вҖ” the simplest possible topology for the P2 masterвҖ“slave elimination and Kutta loop. Coarse/medium/fine family, revised single-layer targets ~15k / 60k / 240k tets (вүҲ 5k/20k/80k 2D triangles Г— 3).
**Gate:**
- [x] Solver preprocessor ingests it; topology asserts pass (see P2); quad-split consistency assert passes; mesh family generated by one script, one parameter (h). Gate G2.5 (quasi-2D spanwise-zero) is the acceptance link between Track M and Track P.
**Visual test examples:**
- VM0.1 render mesh tags with a headless script (PyVista or ParaView batch) and verify `wake` is one continuous internal sheet from TE to far field, `symmetry` planes are planar and non-overlapping with `wall`.
- VM0.2 show element size (or edge-length histogram) for coarse/medium/fine on one figure; verify monotone refinement and similar growth ratio away from airfoil/wake.
**Notes:** doubles as the low-cost rapid test environment for P2вҖ“P4 and the

### P7 вҖ” Differentiable artificial-density flux at frozen selection (Newton prerequisite)

> **Re-scoped 2026-07-08 (was "ship the streamline-Gaussian kernel").** Following
> the analysis to its end: the Newton prerequisite is **not** a new flux вҖ” it is
> the *existing walk flux made differentiable at frozen selection*. The kernel is
> demoted to an optional Picard-speed path. See the motivation.

**Motivation.** The exact Newton Jacobian (P8, design.md В§6.3 / LГіpez
Appendix B) needs вҲӮПҒМғ/вҲӮПҶ **at frozen upstream selection** вҖ” LГіpez re-selects
u(e) each Newton step but never differentiates the selection (design.md В§3.1).
At frozen u(e) the shipped P4 **walk** flux is *already* differentiable: ПҒ_e and
ПҒ_up follow the CВ№ isentropic law, and the switch derivative вҲӮОҪ/вҲӮПҶ exists on each
branch (LГіpez B.3вҖ“B.6). Its only kinks are the two `max` вҖ”
`ОҪ_e = CВ·max(0, 1вҲ’M_cВІ/M_eВІ)` at M_e = M_c and the shock-point `max(ОҪ_e, ОҪ_up)` вҖ”
and **LГіpez keeps exactly this CвҒ° switch and still converges quadratically** (the
active set freezes near the solution; design.md В§3.1). **So the Newton
prerequisite is small and sparse: derive + finite-difference-verify вҲӮПҒМғ/вҲӮПҶ for
the shipped walk flux at frozen selection вҖ” the walk is kept, not replaced.** Its
upstream coupling stays ~+1 element/row (the walk's single u(e)) вҖ” the sparse
structure closest to LГіpez's one-hop stencil. This phase does **not** touch the
sawtooth (a recovery artifact, P6).

**Optional `max_Оө` (robustness only, not required for correctness).** A smooth
`max_Оө` on the **inner** `max(ОҪ_e, ОҪ_up)` removes Jacobian chatter if the active
set churns *far* from the solution during load stepping; it is not needed for
terminal quadratic convergence (LГіpez's CвҒ° `max` suffices). **Never smooth the
outer `max(0, В·)`** вҖ” that clamp is what makes ОҪ вүЎ 0 (ПҒМғ = ПҒ, bit-identical to
P3) subcritically; smoothing it would leak ОҪ into subsonic cells and break
G4.2/G7.1. The outer clamp's kink is inactive at the converged state (0
floored/limited on P5), so it never enters the converged Jacobian.

**Kernel demoted to an optional Picard-speed path (2026-07-08).** The multi-ring
streamline-Gaussian kernel built during P6/N1

вҲҮПҶ, ~10Г— faster per density iteration than the walk вҖ” `tests/test_p6_weighted_flux.py`,
