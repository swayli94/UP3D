**性质**：横向分析报告（非 gate 文档，不改变任何 roadmap 状态；最终裁决 = 用户）。
执行 Kimi 第二轮审查 §3 第 6 条的"P11 优先级重估"建议，并综合 B18–B21 之后的
证据格局。快照日期 2026-07-19；引用数字以各 track 台账 + committed CSV 为准。

## 0. 结论先行（推荐排序）

1. **P11 曲面壁元（Option C gate 重定规格 + curved elements）— 推荐第一优先。**
   它是唯一攻击"随加密恶化"类误差的候选，且 B18/B20 之后其受益面从 1 个 gate
   扩大到 3 个已知伤口（§1）。
2. **附带的便宜 A-track 小项：M_max 跨模型核对**（§4，可与 P11 并行或先行，
   ~1 天）——B21 恢复的 M6 medium M_max 2.4818 至今只有 LS-vs-LS 共模验证。
3. **LS fine 路线（B14 的 designed-not-built 用例）— 第二优先**，在"有具体
   fine 网格问题要回答"时开（§2）；B21 恢复包线后它重新有意义，但当前没有
   一个被阻塞的科学问题非它不可。
4. **Track V 粘性 — 建议放在 P11 之后**（§3）：粘性修正叠在一个已知的、随
   加密恶化的几何离散误差上，会继承并混淆它；先把无粘离散做可信。

## 1. P11 曲面壁元：受益面已从 1 个 gate 扩大到 3 个伤口

G1.6 的根因早已定谳（P1，PROJECT_STRUCTURE "Known gaps"）：**平坦面片壁上的
自然边界条件——变分层面的罪**，h 加密、恢复调参、Nitsche、边界数据修正全部
用 oracle 证据排除（纪律 #8 的 closed negatives），在案唯一路线 = Option C
gate 重定规格 + 曲面壁元。原本这只值一个 sphere gate（11.6% strict xfail），
**现在的证据把三个独立观测钉到同一根因**：

| 伤口 | 证据 | 随加密 |
|---|---|---|
| Sphere 不可压 Cp 11.6% | G1.6 strict xfail（P1） | 收敛到错误答案（变分罪） |
| 翼身机身虚假升力 16–20% wing cl_p | GB9.4 XFAIL（B9/B18：跨声速下仍 16%） | **增长**（0.164→0.205） |
| LS 翼身交界超声速袋 M≈5.2 @M0.5 | GB20.5（解钳后为真实收敛值，非伪影） | **恶化**（coarse Mmax 1.31 → medium 5.22），封死 LS 翼身跨声速（B18 closed-negative） |

关键判别事实（B20/GB20.5，2026-07-19 实测）：把 mixed-plain 污染清干净后交界袋
**没有消失反而露出真身**——它不是数值 bug，是几何离散误差。B21 又排除了 freeze
捕获轴。⇒ **翼身这条线上已经没有"更便宜的嫌疑人"了**：要么接受"LS 翼身止步
M0.5、机身升力带 16–20% 伪影"为长期能力边界，要么开 P11。

可能的顺带收益（无证据，仅列为待验证假设，勿当结论引用）：conforming 翼身
medium M0.80+ 停滞（B18 记录为"更锐激波/交界相互作用，非 sliver"）发生在同一
交界区，曲面壁元**可能**缓解——B18 时未归因，P11 开题时应列为观测项。

成本与风险：壁面函数空间级改动（P1 时代已界定），预计是 Track P 最重的单相位
之一；风险在 Kutta/wake 与曲面元的耦合处。**先做 sphere（无尾迹、oracle 齐全、
gate 已重定规格的路线在案），拿到 G1.6 关闭后再动翼身**，风险可控。

## 2. LS fine 路线：重新有意义，但缺一个非它不可的问题

B14 的诚实边界写得很清楚：Schur+AMG 在 medium 是 1.43–2.08×，**设计价值在
fine 的内存受限路径（AMG O(n) + 薄带 LU，无全尺寸 splu）——没建**。B21 恢复
M6 medium M0.84 之后这条路线重新有意义（GB20.7 时代"包线塌了还谈什么 fine"
的顾虑已消除）。

但要诚实地问：**哪个悬而未决的问题需要 LS fine？**

- 跨模型 fine 对照？conforming fine M0.84 真离散解已有（44.6 min，M_max 2.818），
  但 LS-vs-conforming 的离散差在 coarse 就有 −7.4%（γ，NACA M0.80，用户裁决
  不追）——fine 对照在追它之前意义有限。
- 网格收敛（Richardson）？G13.3 跨声速 Richardson 在 conforming 侧就是
  NEGATIVE-open（几何/奇点问题，不是规模问题）；LS 侧没有阶梯声称待验证。
- 翼身 fine？交界袋随加密恶化（§1）——在 P11 之前，fine 只会把袋放大。

⇒ 排第二：**技术上成熟（B14 已把 Schur 底座建好）、开销中等、但当前没有被它
阻塞的结论**。若用户有 fine 级的具体目标（如 AGARD 定量对照），可提前。

## 3. Track V 粘性：物理收益最大，但时机在 P11 之后

支持先做 V 的论据：项目目标（M∞ 0.3–0.87 机翼）里无粘 FP 对实验的系统性升力
偏高只有粘性修正能治（track_v 设计：Drela IBL3 + transpiration BC，VII 方向
CL 0.245→0.288 的差距语境）；设计完备、预算约一个 Track-P 相位；与 G1.6 正交。

反对现在做的论据（我认为更重）：
1. **误差叠加次序**——粘性修正叠在随加密恶化的几何误差上（§1 的三个伤口），
   翼身与 tip 区域的 transpiration 源项会直接坐在污染的 Cp 上，V 的验证
   （对实验）将无法把"粘性模型误差"与"G1.6 几何误差"分开——正是 P14 教训
   （"别把上一相位的归因带进新测量"）的相位级版本。
2. V1 的自然验证阶梯（2-D/2.5-D 翼型边界层）不受 G1.6 影响，确实可以先走
   V1——若用户想并行，**V1（2.5-D IBL 阶梯）与 P11（sphere 腿）无资源冲突
   之外的耦合**，是可行的并行组合。

## 4. 便宜小项（可插队，均 ≤1–2 天）

- **M_max 跨模型核对**：B21 的 medium M_max 2.4818 与 conforming 记录 1.995
  差 24%，但两者在**不同网格族**（M4 wake-free vs M1 wake-embedded）且 M_max
  是逐点量、对局部网格敏感。一个 A-track 小项：在同族网格上（M1 embedded 也
  跑 LS ramp，或 conforming 跑 M4）对齐后再比，才能知道这 24% 是离散差还是
  族差。**在此之前不要引用"LS M_max 偏高 25%"作为缺陷**（GB20.7 时代的这个
  说法带着"旧状态可疑"的叙事，B21 后需要重新定性）。
- **N3 锁已在 B22 落地**（M6 coarse/medium ramp 锚测试）；下一次 re-baseline
  会有报警。
- G10.1（非升力 Newton 入口）：P10 的收尾卫生项，无顺序约束，随手可做。
