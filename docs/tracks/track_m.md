## Track M ‚ÄĒ Mesh generation pipeline (parallel to solver track)

Deliverable: `meshgen/` ‚ÄĒ scripted Gmsh generation of wake-embedded (M0/M1)
and wake-free (M3/M4, for Track B) tet meshes, coarse/medium/fine families per
case.

**Track-wide convention.** Model the wake as a Gmsh *embedded surface in volume*
(`gmsh.model.mesh.embed`); node duplication is NOT done in Gmsh ‚ÄĒ it is done by
the solver-side preprocessor (`mesh/wake_cut.py`), keeping meshing vanilla
(hard rule 8). The wake-**free** families (M3/M4) drop the embed step entirely:
the wake sheet survives only as the source of a size field, so nothing in the
mesh topology knows the wake exists ‚ÄĒ exactly the form Track B's level set
consumes (dual-mesh rule, Track B working rules). Any wake-cut change reruns the
topology asserts on all meshes in `cases/meshes/` (hard rule 7).

### M0 ‚ÄĒ Single-layer extruded NACA0012 quasi-2D mesh ‚úď (closed 2026-07-06, with P2)
**Deliverable:** **Single-layer extruded NACA0012 quasi-2D mesh.** Input: 2D triangulated airfoil mesh (Gmsh). Extrude exactly ONE cell layer in z between two parallel planes; both z-planes tagged `symmetry`, other surfaces tagged `wall`, `farfield`, `wake`. Prisms subdivided into 3 tets each with a **globally consistent rule**: on every shared quad face, the diagonal connects to the smallest global node index; preprocessor assert that every interior quad face is split identically from both sides (no hanging nodes / slivers). **Planar wake surface embedded** from TE to far field (`gmsh.model.mesh.embed`; node duplication stays solver-side, unchanged). With a single layer there is exactly one spanwise station, so őď is a single scalar ‚ÄĒ the simplest possible topology for the P2 master‚Äďslave elimination and Kutta loop. Coarse/medium/fine family, revised single-layer targets ~15k / 60k / 240k tets (‚Čą 5k/20k/80k 2D triangles √ó 3).
**Gate:**
- [x] Solver preprocessor ingests it; topology asserts pass (see P2); quad-split consistency assert passes; mesh family generated by one script, one parameter (h). Gate G2.5 (quasi-2D spanwise-zero) is the acceptance link between Track M and Track P.
**Visual test examples:**
- VM0.1 render mesh tags with a headless script (PyVista or ParaView batch) and verify `wake` is one continuous internal sheet from TE to far field, `symmetry` planes are planar and non-overlapping with `wall`.
- VM0.2 show element size (or edge-length histogram) for coarse/medium/fine on one figure; verify monotone refinement and similar growth ratio away from airfoil/wake.
**Notes:** doubles as the low-cost rapid test environment for P2‚ÄďP4 and the
simplest `wake_cut.py` testbed. Delivery record: progress ledger, Track M.

### M2 ‚ÄĒ Wing-body: wake‚Äďfuselage junction ‚óź (SCHEDULED + mesh delivered 2026-07-13, user-directed)
**Deliverable:** **ONERA M6 wing + simplified axisymmetric fuselage, fused into
a wing-body half model, meshed WAKE-FREE for the level-set (Track B) path.**
The 2026-07-10 note said the wake‚Äďfuselage intersection is the case the
conforming-sheet approach handles worst, and that M2 should be scheduled with
Track B's embedded wake, which removes the need to pre-embed a sheet. That is
exactly what this does: **nothing is fragmented or embedded** ‚ÄĒ the wake exists
only as a corridor size field, and the solver builds it from the analytic TE
polyline via `WakeLevelSet`. The mesh has **no `wake` group**.

`meshgen/fuselage.py` (`FuselageParams`, `radius_at`, `add_fuselage_solid`) +
`meshgen/wingbody.py` (`onera_m6_wingbody_mesh`, `te_polyline`, `junction_z`).
`wing3d.py` is **byte-untouched** ‚ÄĒ the wing loft, the M5 round cap and
`_collect_3d` are imported and reused, so every existing M1/M4/M5 family stays
bit-identical.

‚ėÖ **The fuselage is ONE splined body of revolution, not four fused primitives.**
Fusing nose-ellipsoid + cylinder + cone + tail-sphere leaves a C0 seam at every
junction (the cylinder‚Üícone shoulder especially), and a surface crease is a
spurious edge feature ‚ÄĒ the flat-tip-cap problem M5 exists to remove. A single
meridian profile splined through the same piecewise radius law and revolved
about the chord axis gives a C2 skin, so the only sharp wall features on the
whole wing-body are the wing LE/TE, which carry the physics.

‚ėÖ **`junction_z = r_f` is EXACT, not approximate:** the M6 root chord
(x ‚ąą [0, 0.806]) lies wholly inside the constant-radius section, and the wing TE
sits at y = 0 (the ONERA D section has zero TE thickness), so the TE line meets
the fuselage skin at z = r_f exactly. That vertex is what the level-set wake's
inboard end attaches to.

**Gate:**

Quality inside the M1 bounds (min dihedral **5.65¬į / 6.34¬į**, max aspect **11.6 / 6.4**). Wake-free (no `wake` group); boundary groups `wall` (wing skin) / `fuselage` / `farfield` / `symmetry`.
- [x] ‚ėÖ **THE GATE METRIC IS THE FUSELAGE CREASE ANGLE** (`post/surface.py::wall_crease_angles`, the M5 argument applied to the body ‚ÄĒ the solver only ever sees the TRIANGULATION, so "the CAD is smooth" is not the claim that matters). p99 **20.6¬į ‚Üí 11.2¬į, q ‚Čą ‚ąí0.88 ‚áí O(h)**; median **3.60¬į ‚Üí 0.80¬į**. It **DECAYS** ‚áí faceting of a smooth surface ‚áí **no seam in the limit**. (Contrast M5's flat cap, a real edge: 91.9¬į ‚Üí 92.1¬į, q = ‚ąí0.00 ‚ÄĒ *does not move*.) ### M5 ‚ÄĒ Rounded ONERA M6 tip cap ‚úď (NEW 2026-07-13, user-directed; closed 2026-07-13)
**Deliverable:** **The M6 tip cap, rounded ‚ÄĒ the geometry fix P13/G13.3 asked
for.** M1 closes the wing with a **flat** tip cap: a documented deliberate
simplification, standard for FP validation meshes, and the third and last of the
defects that blocked a three-point Richardson on ONERA M6. A flat cap meets the
upper/lower surfaces at a **sharp convex edge**, and in potential flow a sharp
convex edge is a singularity ‚ÄĒ G13.3's box study measured its peak Mach
**diverging** under uniform refinement (p = +0.321) while the wake free edge
(+0.045, fixed by G13.2) and the wing interior (‚ąí0.014) stayed bounded. It is
**not a P11 problem**: isoparametric *elements* cannot regularize a genuinely
sharp geometric *edge*. The geometry is what is wrong.

`meshgen/wing3d.py` gains **`tip_cap="round"`** (default `"flat"` ‚áí every
existing family bit-identical, so the P5 / P8-G8.2 / B7 / M1 locks are
untouched): the wing is closed by the **half body of revolution swept by the tip
section about its own chord line**, i.e. `{‚ąö(y¬≤ + (z‚ąíB_SEMI)¬≤) ‚Č§ t(x)}` with
`t(x)` the tip section's local half thickness. Built as an OCC revolve of the
tip section's upper half-face about an edge OF that face (the degenerate-at-the-
axis kind, like a sphere from a half disc), fused onto the loft.

‚ėÖ **Why this construction and not another:** the cap radius **vanishes at the LE
and the TE**, so the cap degenerates to a *point* at each ‚ÄĒ which means the **TE
line, the wake sheet, the tip TE corner, the Kutta stations and B_SEMI are all
UNCHANGED**. Only the tip WALL moves. That is what makes the A/B against M1
controlled, and it is why no solver-side plumbing was needed. (Verified
analytically and asserted at generation: the revolved solid protrudes past the
wing by at most **4e-6 m** ‚ÄĒ three orders below the finest edge size ‚ÄĒ and never
reaches aft of the local TE, so it can neither cut the wake sheet nor spawn
slivers.)

**Gate:**
- [x] **CLOSED 2026-07-13** (`cases/meshes/onera_m6_roundtip/generate_onera_m6_roundtip.py`; demo `cases/demo/m5_round_tip/` **9/9 PASS**; `tests/test_m5_round_tip.py` **19**). ‚ėÖ **The gate metric is the SEAM CREASE ANGLE** ‚ÄĒ the turning angle between adjacent wall triangles across the tip-section seam at z = B_SEMI, i.e. the locus that *is* the sharp edge in M1 (`post/surface.py::wall_crease_angles`, measured away from the LE/TE, which are sharp **by design** in both families ‚ÄĒ the TE carries the Kutta condition). The solver only ever sees the **triangulation**, so "the CAD is round" is not the claim that matters. Over the full three-level ladder (p99): **flat 91.9¬į ‚Üí 92.1¬į ‚Üí 92.1¬į, exponent q = ‚ąí0.00** ‚ÄĒ *does not move*; a real edge, refinement resolves it and removes nothing ‚ÄĒ vs **round 46.8¬į ‚Üí 24.5¬į ‚Üí 13.7¬į, q = ‚ąí0.92** ‚áí **O(h)** faceting of a smooth surface ‚áí no edge in the limit. (Honesty: the flat cap's *median* seam crease is **0.00¬į**, i.e. this metric reads the edge and nothing else; and the round cap's *max* decays more slowly, 46.8 ‚Üí 25.0 ‚Üí 18.1¬į, q = ‚ąí0.68, because the max is one worst facet at the thin end of the seam window where the facet size is set by `h_edge`, not `h_tip`. Both DECAY, which is the claim.) Cap resolved to its apex (wall z_max 1.218465/1.218466/1.218467 vs the analytic apex 1.218467). Quality inside the M1 bounds (min dihedral 4.05¬į/5.18¬į/8.63¬į, max aspect 16.2/6.4/6.4). `cut_wake` keeps the M1 semantics (85 TE stations, tip TE corner still a free-edge node ‚áí őď(tip)=0 discretely); G2.1 freestream on the cut mesh < 1e-10. **Nothing else moved**: tip TE corner offset **0.0**, wake sheet still in the chord plane and ending at B_SEMI.
- [x] **Ladder self-similar from the start** ‚ÄĒ `RICHARDSON_LADDER = (coarse, medium, fine)` with **no h_far clamp** (the M1b defect is guarded against by a test) and **`h_tip = 0.25¬∑h_wall`** so the cap refines with everything else. The cap radius is only 22 mm (1.9% of the semi-span), so at `h_wall` the coarse cap would be about one element wide and would **discretize back into a flat one** ‚ÄĒ h_tip is what stops the geometry change from being silently undone by the mesh. **COST: √ó1.29 / √ó1.28 / √ó1.30 tets** (59,359 / 448,197 / 3,257,273 vs M1's coarse_ss 46,067 / medium 350,718 / fine 2,513,255) ‚ÄĒ level-independent, i.e. the refinement ray is preserved.
**Notes:** `.msh` gitignored (M1 policy); stats CSVs + inspection PNGs committed
as the evidence record. Consumed by P13/G13.3
