# pyFP3D — Copilot Development Instructions

## Overview

This is a **3D unstructured-mesh full-potential transonic flow solver** in Python + Numba.
Target: steady external flow over wings at Mach 0.3–0.87, fully transonic, isentropic inviscid.

**Key distinction:** Full-potential (FP) replaces Euler's 5 conservative vars with **one scalar φ (velocity potential)** → 1–2 orders cheaper per solve, ideal for optimization and surrogate loops.

Authoritative reference: [docs/design.md](docs/design.md)  
Active roadmap: [docs/roadmap.md](docs/roadmap.md)  
Working rules: [docs/agent-rules.md](docs/agent-rules.md)

## Phase Hierarchy and Gating Strategy

Development is organized in **phases P0–P5** (solver) and **M0–M2** (meshing).
**A phase closes ONLY when:**
1. Medium-mesh gate passes (marked ☑ in roadmap)
2. Coarse regression suite stays green
3. All visual checks have headless PNG + CSV artifacts

**Golden rule:** Do NOT advance to the next phase until the gate passes.

## P1 — Laplace Solver (ρ ≡ 1)

### Scope
Incompressible (ρ=1) potential flow. Sets up all linear-algebra infrastructure: Galerkin weak form, AMG preconditioner, CG solver.

### Key physics

The **quasi-linear elliptic operator** (subsonic everywhere since ρ=1):
```
∇²φ = 0  (Laplace equation)
```

**Weak form (Galerkin):**
```
∫_Ω ∇φ · ∇v dV = ∫_∂Ω_N g v dS   (Dirichlet on far field, natural on walls)
```

### Numerical infrastructure

1. **Residual assembly:** Element-wise (tetrahedral basis, linear elements)
   - Input: nodal potentials φ_i
   - Output: global residual vector and Jacobian matrix (for Newton, though P1 is single-linear-solve)

2. **Boundary conditions:**
   - Far field: Dirichlet φ = x (freestream reference)
   - Walls: Neumann (natural, zero normal flux by default)

3. **Linear solver:**
   - CG + PyAMG smoothed aggregation preconditioner
   - Iteration tolerance: 1e−8 relative residual

### Acceptance gates

**G1.1: Method of Manufactured Solutions (MMS) convergence**
- **Setup:** Exact solution φ_exact(x,y,z) and source term f
- **Criterion:** L² error convergence slope ≥ 1.9 over 3 mesh levels (coarse, medium, fine)
- **Visual artifact:** `artifacts/G1.1/mms_convergence_loglog.png` (h vs L² error with best-fit slope)

**G1.2: Incompressible sphere**
- **Test:** Uniform flow around unit sphere, Dirichlet BC φ = x at farfield
- **Exact solution:** φ = x + (1/2r³)(x) with r = √(x²+y²+z²)
- **Criterion:** Max |Cp − (1 − 9/4 sin²θ)| < 2% on medium mesh
- **Implementation:** `tests/test_laplace_sphere.py` (not written yet)
- **Visual artifacts:**
  - `artifacts/G1.2/sphere_cp_meridian.png` (line plot: computed Cp vs θ, overlaid with exact curve)
  - `artifacts/G1.2/sphere_cp_contour.png` (contour of Cp on sphere surface)
- **Status:** `cases/meshes/sphere_shell/{coarse,medium}.msh` and the solver path
  (`solve_laplace` + `post/surface.py::nodal_gradient_recovery`) are in place, but a manual run
  against the medium mesh gives ~26% max Cp error (vs. the 2% target) — `nodal_gradient_recovery`'s
  one-sided volume-weighted average is not accurate enough at the curved wall boundary yet. Fix
  that before writing this test.

**G1.3: Mesh-independent iteration count**
- **Test:** Run CG + AMG on coarse/medium/fine Laplace problems
- **Criterion:** Iteration counts differ by <5% across mesh levels
- **Visual artifact:** `artifacts/G1.3/cg_convergence_overlay.png` (semi-log plot of CG residual vs iteration for 3 meshes)

## P2 — Wake Cut + Circulation + Kutta Condition ★ CRITICAL PHASE

### Why this phase is critical

- **Wake topology:** First time we handle multivalued potential (lifting flow)
- **Node duplication:** Physical and logical: φ⁺ ≠ φ⁻ across wake
- **Constraints:** Master–slave DOF elimination (φ⁺ = φ⁻ + Γ)
- **Kutta enforcement:** Ensures physically correct pressure jump at trailing edge

### Mesh requirement

The **wake surface must be a conforming internal mesh surface:**
- Tagged (e.g., `wake` in Gmsh)
- Elements on either side attach to duplicated nodes (i⁺, i⁻)
- Extends from trailing edge TE to far field
- No self-intersections or cracks

Preprocessing (solver-side) duplicates nodes; keeps mesh generation vanilla.

### Topology asserts (run at every mesh preprocessor call)

```python
def validate_wake_topology(mesh):
    # Every interior wake face has exactly one '+'-side and one '−'-side element
    # TE nodes are NOT duplicated
    # No orphan or dangling duplicated nodes
    # Duplicated node count == interior wake face nodes
    # Symmetry-plane edges don't accidentally alias wake edges
```

### Acceptance gates

**G2.1: Freestream with cut**
- **Test:** φ = x on the **cut mesh** (Γ = 0), verify residual
- **Criterion:** ‖R‖∞ < 1e−12 and ‖R‖₂ < 1e−11
- **Why:** Fundamental test that wake-cut topology doesn't break consistency
- **Artifact:** `artifacts/G2.1/freestream_residual_heatmap.png` (per-node |R_i|, expect white noise)

**G2.2: Jump condition consistency**
- **Test:** Prescribe Γ = 0.5 (constant), verify [φ] = Γ on all wake faces
- **Criterion:** Max |[φ] − Γ| < 1e−10
- **Artifact:** `artifacts/G2.2/jump_check_histogram.png` (histogram of [φ] − Γ across wake)

**G2.3: Incompressible lifting (NACA0012 extruded)**
- **Test:** Lift from extruded NACA0012, Laplace solver with Kutta update
- **Criterion:** CL matches inviscid theory for small α; trend matches literature
- **Artifact:** `artifacts/G2.3/lift_vs_alpha.png` (CL vs angle of attack)

## P3 — Isentropic Density + Artificial Compressibility

### Transonic treatment

Replace ρ in residual assembly with an **upwinded density** to suppress expansion shocks:
```
ρ̃ = ρ − ν · Δℓ · (∂ρ/∂ℓ)_upwind
ν = C · max(0, 1 − M_c² / M²)
```

Parameters: C ∈ [1, 2], M_c ≈ 0.95–1.0. Expose in config from day 1.

### Acceptance gates

**G3.1: Subsonic limit**
- **Test:** Run on incompressible sphere test (P1.2), rerun with isentropic ρ but M∞ → 0
- **Criterion:** Cp error < 1% (should recover Laplace limit)

**G3.2: Transonic shock capture**
- **Test:** NACA0012 at M∞ = 0.8, α = 2°
- **Criterion:** Shock within 2 elements of experiment; monotone capture (no overshoot)
- **Artifact:** `artifacts/G3.2/shock_mach_profile.png` (M contour with shock band)

## Critical Numerical Constraints (All Phases)

### 1. Precision and Tolerance Strategy

| Quantity | Tolerance | Context |
|----------|-----------|---------|
| Volume conservation | 1e−12 | G0.1 |
| Gradient on linear fields | 1e−14 | G0.2 |
| Freestream residual | 1e−12 (∞), 1e−11 (L₂) | G2.1 |
| Wake jump [φ] | 1e−10 | G2.2 |
| Linear solver | 1e−8 relative residual | CG default |
| Nonlinear Newton | 1e−6 relative step | Default iterate |

### 2. Conservation Laws to Preserve

- **Volume:** Σ V_e = exact domain volume (enforce in G0.1)
- **Mass flux:** ∫_∂Ω ρu_n dS = 0 (implicit in natural BC)
- **Circulation:** ∮ u·dl = Γ on wake (enforce via [φ] = Γ constraint)
- **Freestream:** φ = x + O(1/r) at ∞ (Dirichlet BC)

### 3. Physics Bounds

Always validate:
```python
def validate_physics(rho, q, M, Cp, M_inf, gamma=1.4):
    # ρ ∈ (0, ρ₀]  (density must be positive)
    # q ∈ [0, 1 + ε]  (speed ≤ local sound speed; ε ~ 1%)
    # M ∈ [0, 5]  (Mach number reasonable range)
    # −5 ≤ Cp ≤ 5  (pressure coefficient bounded)
    # Critical speed q* = √[ (2 + (γ−1)M∞²) / ((γ+1)M∞²) ]
    assert all(rho > 0), "Negative density!"
    assert all(q <= 1.01), f"Supersonic speed q={max(q)} exceeds c"
    # ...
```

## Numba Optimization Rules (Hard Rules)

1. **Structure-of-Arrays (SoA) only** in @njit kernels
   - NO Python objects inside @njit
   - NO dynamic lists; pre-allocate fixed-size arrays
   - NO dict lookups in hot loops

2. **@prange for element loops**
   - Color the element graph → issue independent colors to threads
   - Residual += (not =) to avoid race conditions on shared nodes
   - One accumulation pass per color → no lock overhead

3. **Zero allocation in hot loops**
   - Pre-allocate working arrays outside loop
   - Reuse temporary storage across iterations

4. **cache=True on @njit**
   ```python
   @numba.njit(cache=True)
   def assembly_kernel(...):
       ...
   ```

5. **PYFP3D_NOJIT environment variable**
   - For debugging: `export PYFP3D_NOJIT=1` → run pure Python version
   - Allows print(), pdb, traceback in development

## Physics Constants and Reference Data

All physics scalars belong in **`pyfp3d/physics/isentropic.py`**:

```python
GAMMA = 1.4                 # Specific heat ratio (air)
R_GAS = 287.05              # Gas constant (J/kg·K)
# ...

@numba.njit
def isentropic_density(q_sq, M_inf, gamma=GAMMA):
    """Return ρ(q²) from isentropic law."""
    return (1.0 + 0.5 * (gamma - 1) * M_inf**2 * (1 - q_sq))**(1 / (gamma - 1))

@numba.njit
def critical_speed_squared(M_inf, gamma=GAMMA):
    """Return q*² where M = 1."""
    return (2 + (gamma - 1) * M_inf**2) / ((gamma + 1) * M_inf**2)

# Unit test every function against hand values
```

## Common Pitfalls and Debugging Checklist

### Mesh issues
- [ ] Wake nodes properly duplicated (i⁺ ≠ i⁻)
- [ ] Wake surface is conforming (no cracks, T-junctions, or orphan elements)
- [ ] Boundary tags match expected names (`wall`, `farfield`, `symmetry`, `wake`)
- [ ] Element ordering is consistent (no negative-volume elements)

### Numerics
- [ ] Freestream residual ‖R(φ=x)‖ < 1e−12 **BEFORE** advancing phases
- [ ] Matrix is SPD in subsonic zones (verify eigenvalues if Newton stalls)
- [ ] Density upwinding is stable (shock should not oscillate)
- [ ] CG convergence should be monotone; if erratic → preconditioner issue

### I/O
- [ ] VTK export preserves all nodal/cell fields (spot-check a few values)
- [ ] Mesh round-trip (read → write → read) is bit-identical
- [ ] Reference data (Cp tables, analytic solutions) are committed to `reference_data/` (immutable)

### Parallelization
- [ ] Element coloring is valid (test in G0.3)
- [ ] @prange reductions use correct accumulation (+=, not =)
- [ ] No data races on shared nodes (use color-ordering)
